1. 기초
    1) 함수를 이용하여 프로그램을 기능 단위로 나누는 방식을 구조화 프로그래밍이라고 한다.

2. 이름 규칙 
    1) 제일 앞 글자는 영문 또는 _로 시작한다
    2) 두번째 글자부터는 영문, _, 숫자 등이 올 수있다
    3) 대소문자를 구분한다
    4) 이름 사이에 공백이나 특수문자가 올 수 없다

3. 변수
    1) 전역 변수는 일반적으로 #include 문 아래에 선언한다
    2) 지역변수와 전역변수는 메모리 해제 시점이 다르다

    3) 정적변수 static
        (1) 해당 static 변수를 선언하는 함수의 최초 한번 호출 이외의 변수 선언 및 할당은 무시된다.
            - 보통 프로그램을 종속적이지 않게 하려면 전역변수보다 static을 활용하는게 좋다
        (2) 전역 변수에 static 을 붙이면 같은 소스 파일에서는 접근이 가능하지만
            다른 소스 파일에서는 접근이 불가능하다.

    4) 전역변수와, 정적변수는 초기화하지 않아도 컴파일러가 0으로 초기화 해준다

4. 재귀 함수
    1) 재귀 호출을 구현할 때 반드시 지켜야 하는 규칙
        (1) 종료 조건이 있어야 한다
        (2) 호출될 때마다 문제의 범위가 줄어들어야 한다.
